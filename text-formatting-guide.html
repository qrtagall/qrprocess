TEXT artifact formatting

For FileType TEXT, the spreadsheet columns are Basic Info · FileType · Options · Local Link. In the edit form, Basic Info is the artifact title and Text Info is saved to Local Link. Formatting rules below apply to Local Link / Text Info content. They apply on the public QR page when Options is VIEW.

1. Page layout (title + body)

How the artifact block is arranged on the public page depends on Basic Info and Local Link. These rules apply only to TEXT — not to file uploads or MESSAGEEMAIL.

Title only — Local Link empty Hidden Centered (no link icon) — (no body)

If both “title only” and “headless” could apply (e.g. empty title and empty Local Link), the headless rule wins — an empty block is shown.

2. Label lines

Text before the first : on a line is shown as a bold blue label.

You typeDisplayed as
Phone: 9876543210Phone: 9876543210
Vehicle No: MH12AB1234Vehicle No: MH12AB1234

Up to 10 words before : qualify. Longer prefixes stay plain text. URLs like https:// are not treated as labels. If inline markup already appears in the label part (e.g. "Name": value), label styling is skipped for that line.

3. Inline styles

Use matching pairs only. Plain apostrophes in words (e.g. Owner's) are not styled.

SyntaxResult
"text"bold
'text'italic
`text`monospace
~~text~~strikethrough
==text==highlight
You type
Name: "Chandan Kumar" Plate: `MH12AB1234` Status: ==Active==

4. Processing order (plain lines)

Outside [HTML] blocks, each non-blank line is processed in this order:

  1. Inline styles ("bold", 'italic', …)
  2. Web URLs → smart link cards
  3. Label: styling
  4. Phone numbers → Call / WhatsApp actions

Blank lines outside HTML blocks are ignored.

5. Phone numbers

10-digit Indian-style numbers get Call (📞) and WhatsApp (💬) actions. Leading 0 or +91 is normalised. Other lengths are left as plain text.

Example: Mobile: 9876543210

6. Web links

https://… URLs become compact smart link cards with Open and Copy.

Recognised platforms (icon + colour): YouTube, Facebook, Instagram, LinkedIn, X/Twitter, Google Drive, Google Docs, Google Forms, Google Maps, WhatsApp, and generic web links.

7. Google Drive on a line

Put optional caption text, then a Drive file URL on the same line:

Manual https://drive.google.com/file/d/…

The line text is replaced by a thumbnail gallery at the bottom of the artifact; the caption appears under each thumb. Multiple Drive file lines are combined into one gallery. Use file URLs (/file/d/… or open?id=…) — not the same as a folder gallery (FileType DRIVE).

8. HTML blocks (optional)

Wrap HTML in explicit delimiters. Outside the block, normal rules above still apply. Inside the block, raw HTML is rendered (with safety filtering).

Delimiters

FormUsage
[HTML] … [/HTML]Inline block or entire field (case insensitive)
<html> … </html>Entire field only (case insensitive)

Mixed plain + HTML

You type
Phone: 9876543210 [HTML] <ul> <li><b>Insurance</b> — valid</li> <li><b>PUC</b> — valid</li> </ul> [/HTML] POLICE: Phone: 112

Allowed & blocked in HTML blocks

  • Allowed: headings, paragraphs, lists, tables, links, images, inline styles
  • Removed for safety: <script>, <iframe>, <object>, <embed>, onclick= and other event attributes, javascript: URLs
  • Unclosed block: [HTML] without a matching [/HTML] is treated as plain text (not HTML)
Tip: Use HTML for tables or multi-column layouts. Use plain Label: value lines for simple sticker fields — they are easier to edit on a phone.

9. Line breaks

  • One logical item per line outside HTML blocks
  • Blank lines are ignored outside HTML blocks
  • Inside [HTML] blocks, line breaks in your markup are preserved
  • Plain segments and HTML blocks are separated by a line break when both appear in one field

10. Vehicle sticker example

Sample Local Link content
VEHICLE NO: "MH12AB1234" Name: 'Owner Name' Mobile No: 9876543210 [HTML] <table> <tr><th>Document</th><th>Status</th></tr> <tr><td>Insurance</td><td style="color:green">Active</td></tr> </table> [/HTML] POLICE: Phone: 112

11. What does not apply to TEXT

  • NOVIEW (Options) — hidden from visitors; owner still sees the artifact in edit mode
  • File uploads use PDFFILE, IMAGEFILE, etc. — not TEXT
  • MESSAGEEMAIL is a separate type (Contact owner button)
  • DRIVE / GDRIVE — folder thumbnail gallery from a folder URL; different from Drive file thumbs inside TEXT
  • Layout rules in section 1 apply to the artifact frame; sections 2–10 apply to Local Link content inside the frame
